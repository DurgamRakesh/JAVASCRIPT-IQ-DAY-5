27.What is hoisting?

        IN JS, Hoisting is a javascript mechanism where functions and variables are moved to top of the their scope before code execution.
        -Javascript allows only variable declarations , not initialization.
        -variable hoisting: In terms of variable hoisting , like var is hoisting but let and const keywords are not hoisted because they are in temporal dead zone.
        -we can access a function or a variable before its declarations this is called hoisting behavior.

28.What is a Temporal Dead Zone?

        IN JS, A Temporal Dead Zone is a behavior that occur when declating a variable with let and const keywords , but not happens with var keyword.
        -The time span when that happens, between the creating a variable and binding its declaration is called Temporal Dead Zone.

30.Explain passed by value and passed by reference?

        passed by value:-
        function is called directly by passing a value of a variable as an argument.
        so any changes made outside of the function does not effect the original value.
        -In pass by value, parameters passed as an arguments create its own copy.
        so any changes , made inside function made to copy value not original value.

        EX:
        function sqr(a){
           return a*a
        }
        let b = 10;
        console.log(sqr(b)); --->100
        console.log(b); --->10 value not change

        passed by reference:-
        function is called directly by passing reference/address of a variable as an argument.
        so any changes made outside of the function is effect the original value.
        -In pass by reference, parameters passed as an arguments does not create its own copy.
        so any changes , made inside function is effect to the original value.
        
        EX:
        let student = {
            name:'abc',
            id: 1001
        }
        function incid(obj){
            obj.id+=1
        }
        incid(student);
        consol.log(student);

32.What is the purpose of async/await keywords?

        IN JS, async function is a function declared with the async keyword.
        and await is keyword is used inside async function.
        -The async/await keywords are enables asychronous ,promise-based behavior to written in a cleaner style.avoiding need of promise chain.
        -The use of await keyword pause the function until the promise return the result value.

        EX:
        let p  = new Promise((res,rej) => {
            setTimeout(() => {
                res('this is promis function')
            }, 5000);
        })
        
        async function asyncfunction(){
            let result = await p
            console.log(result);
            console.log('this is async function');
        }
        asyncfunction();
